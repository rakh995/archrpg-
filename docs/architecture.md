# Техническое описание симулятора

### 1. Модель памяти
Симулятор имитирует кэш-память первого уровня (L1). Пользователь может настраивать количество блоков (Capacity), что влияет на частоту вытеснения данных.

### 2. Реализация LRU (Least Recently Used)
Алгоритм реализован через структуру данных "Стек". При каждом обращении к адресу:
- Если адрес в стеке: перемещаем его в вершину (Hit).
- Если адреса нет: добавляем в вершину, при переполнении удаляем нижний элемент (Miss).

### 3. Интеграция LLM (Gemini 1.5 Flash)
ИИ анализирует массив вводимых адресов пользователя и вычисляет коэффициент локальности. На основе этого генерируются персонализированные советы в блоке "Mentor Advice".
