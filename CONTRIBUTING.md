# Правила разработки ArchRPG

Мы придерживались следующих стандартов в ходе проекта:

1. **Ветки (Branches):** Каждая роль работала в своей ветке (`feature/ui`, `feature/logic`, `feature/ai`). Слияние в `main` происходило только после проверки DevOps (Дамир).
2. **Commit Messages:** Использование префиксов `feat:` (новая фича), `fix:` (исправление), `docs:` (документация).
3. **Code Style:** Использование ES6+ стандартов и функциональных компонентов React.
4. **AI Policy:** Gemini использовалась как ассистент для рефакторинга кода и генерации обучающего контента.
